# SecureCodingDemo

Учебное приложение для демонстрации типовых ошибок безопасной разработки в ASP.NET Core.
Каждый пример уязвимости содержит описание проблемы, сценарий атаки, пример уязвимого поведения, безопасный подход и контрольный список для промышленной эксплуатации.

>Для быстрого старта открой `/demo`: там можно выбрать любой сценарий, выполнить готовый `unsafe` и `safe` запрос и сразу сравнить результат. Ссылка `Открыть Markdown раздел` в playground ведет к реальному файлу из каталога `docs`. Для прямой работы с API используй `/swagger`.

## Базовые принципы безопасной разработки

- Не доверять никакому внешнему вводу: данные от клиента должны валидироваться, ограничиваться и кодироваться в нужном контексте.
- Разделять данные и инструкции: SQL, HTML, пути к файлам, цели отражения и источники пакетов не должны строиться из произвольной строки.
- Минимизировать привилегии: пользователи БД, сервисные учетные записи, `JWT`-утверждения, публичные `DTO` и доступ к файловой системе должны иметь только необходимый минимум прав.
- Уважать ресурсные лимиты: размер тела запроса, тайм-ауты, отмена запросов, ограничение частоты и лимиты конкуренции обязательны для публичного API.
- Защищать цепочку поставки: фиксировать версии, проверять предупреждения об уязвимостях, использовать доверенные источники пакетов и хранить секреты вне репозитория.
