# Dangerous File Upload

[← К оглавлению](../README.md#topics)

## Что не так

Загрузка файлов становится уязвимой, когда приложение доверяет типу содержимого от клиента, исходному имени файла и хранит содержимое так, будто оно уже безопасно.

## Типовой уязвимый кейс

Сервер принимает `invoice.pdf`, хотя внутри лежит скрипт, полиглот-файл или исполняемое содержимое.

## Пример атаки

```text
shell.aspx renamed to invoice.pdf
```

## Последствия

- Выполнение кода при небезопасной выдаче файла.
- Хранение вредоносного содержимого.
- Сохраненный `XSS` и перезапись существующих файлов.

## Как исправлять

- Проверять допустимые расширения и сигнатуры файла.
- Генерировать серверное имя файла.
- Хранить загрузки вне веб-каталога и отдавать их через отдельный обработчик скачивания.

## Контрольный список

- Добавить антивирусную проверку или конвейер обезвреживания содержимого.
- Ограничивать размер и тип файла.
- Сохранять метаданные файла и результат проверки отдельно от самого содержимого.
