# Unsafe Reflection

[← К оглавлению](../README.md#topics)

## Что не так

Отражение становится опасным, когда недоверенный ввод управляет именем метода, типа или члена сборки. Тогда клиент получает доступ к внутренним действиям, которых не было в публичном контракте API.

## Типовой уязвимый кейс

API принимает `methodName` и через отражение вызывает любой открытый метод сервисного класса.

## Пример атаки

```json
{"methodName":"DropUserSessions"}
```

## Последствия

- Вызов непредусмотренных ветвей кода.
- Обход контракта API, проверяемого на этапе компиляции.
- Рост поверхности атаки без явного изменения публичного API.

## Как исправлять

- Заменить диспетчеризацию через отражение на обработчики из списка допустимых значений или карту команд.
- Разделять внутренние методы и публичные действия API.
- Ограничивать использование отражения конфигурацией и авторизацией, если без него нельзя.

## Контрольный список

- Проверять места с `GetMethod`, `Invoke` и `Activator`.
- Запретить отражение по клиентским строкам в стандартах разработки.
- Добавить тесты на попытки вызвать неразрешенные операции.
