# XSS

[← К оглавлению](../README.md#topics)

## Что не так

XSS возникает, когда приложение вставляет пользовательский ввод в HTML без контекстного кодирования. Браузер воспринимает строку как разметку и выполняет активное содержимое в доверенном контексте приложения.

## Типовой уязвимый кейс

Приложение выводит комментарий как `<div>{input}</div>`, не кодируя `<script>`, обработчики событий и другие HTML-конструкции.

## Пример атаки

```text
<img src=x onerror=alert(document.domain)>
```

## Последствия

- Кража cookie и маркера доступа из браузерного контекста.
- Выполнение действий от имени пользователя.
- Подмена интерфейса и фишинг внутри доверенной страницы.

## Как исправлять

- Для обычного текста всегда применять HTML-кодирование.
- Для ограниченного HTML использовать очистку по списку допустимых тегов и атрибутов.
- Добавлять `CSP` как дополнительный барьер, а не как замену исправлению.

## Контрольный список

- Включить `HttpOnly`, `SameSite` и `Secure` для cookie.
- Проверить все места, где формируется HTML.
- Добавить автотесты на опасные полезные нагрузки и сценарии сохраненного `XSS`.
