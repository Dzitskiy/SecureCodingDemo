## Что не так

Слабая валидация проверяет только наличие допустимого фрагмента, а не весь формат значения. Поэтому `../../secret.txt` легко проходит проверки вида `Contains(".txt")`.

## Типовой уязвимый кейс

API принимает имя файла, но на деле допускает разделители пути, обход в кодированном виде и служебные суффиксы.

## Пример атаки

```text
..\..\private\secrets.txt
```

## Последствия

- Выход за пределы каталога и чтение файлов вне разрешенной области.
- Передача вредоносных значений в файловую систему, командную оболочку или зависимые сервисы.

## Как исправлять

- Описывать допустимый формат через регулярное выражение, которое проверяет все значение целиком.
- Нормализовать путь после валидации.
- Проверять, что итоговая цель остается внутри доверенного корневого каталога.

## Контрольный список

- Не смешивать имя файла и путь в одном поле.
- Журналировать отклоненные значения как признаки возможной атаки.
- Покрывать тестами полезные нагрузки с кодированным обходом каталога.
