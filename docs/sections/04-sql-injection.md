# SQL Injection

## Что не так

Если пользовательский ввод попадает в SQL как часть синтаксиса, атакующий может менять `WHERE`, `UNION`, `ORDER BY` и даже выполнять destructive statements.

## Типовой уязвимый кейс

Запрос строится строковой интерполяцией: `... where username = '{username}'`.

## Payload

```sql
' OR '1'='1
```

## Последствия

- Обход аутентификации.
- Утечка данных через UNION и blind techniques.
- Изменение или удаление данных.

## Как исправлять

- Использовать parameterized queries.
- Ограничивать права DB account.
- Для динамических структурных частей применять allowlist, а не строковую конкатенацию.

## Production checklist

- CI-проверки на raw SQL hotspots.
- Read-only роли там, где запись не нужна.
- Мониторинг аномальных SQL patterns и ошибок парсинга.
