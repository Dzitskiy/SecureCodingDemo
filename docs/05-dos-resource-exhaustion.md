# DoS / Resource Exhaustion

[← К оглавлению](../README.md#topics)

## Что не так

Переполнение памяти и нехватка потоков чаще всего появляются из-за полного чтения тела запроса в память, долгих синхронных операций и игнорирования отмены.

## Типовой уязвимый кейс

API читает огромный `JSON` целиком в строку, затем выполняет блокирующую обработку в рабочем потоке.

## Пример атаки

```json
{"items":["AAAA.... repeated millions of times"]}
```

## Последствия

- Рост памяти и нагрузки на сборщик мусора.
- Нехватка потоков пула и скачок задержек.
- Перезапуски процесса или деградация соседних конечных точек.

## Как исправлять

- Ограничивать размер тела запроса.
- Использовать потоковую десериализацию и поэтапную обработку.
- Прекращать работу по `CancellationToken`.

## Контрольный список

- Настроить квоты на входящие запросы.
- Задать тайм-ауты и механизмы обратного давления для зависимых вызовов.
- Проводить отдельные нагрузочные тесты на слишком крупный ввод.
