## Что не так

`Path.Combine` не делает путь безопасным. Если пользователь контролирует относительный путь, он может выйти из допустимого каталога через `../` и его варианты.

## Типовой уязвимый кейс

API читает файл из каталога `public`, но принимает `../private/secrets.txt`.

## Пример атаки

```text
../private/secrets.txt
```

## Последствия

- Произвольное чтение файлов.
- Утечка секретов, конфигурации и приватных ключей.
- Дальнейшая компрометация по найденным учетным данным.

## Как исправлять

- Получать канонический базовый путь.
- Нормализовать итоговую цель.
- Проверять, что итоговый путь начинается с доверенного корневого каталога.

## Контрольный список

- Предпочитать идентификаторы документов вместо прямого ввода пути.
- Покрывать тестами кодированный обход и смешанные разделители.
- Убирать прямой доступ к файловой системе из публичных API там, где это возможно.
